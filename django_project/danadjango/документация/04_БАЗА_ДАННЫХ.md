## 📊 Структура базы данных

### 1. accounts_customuser (Пользователи)

Таблица пользователей системы

| Поле | Тип | Описание |
|------|-----|----------|
| id | bigint | Первичный ключ |
| username | varchar(150) | Имя пользователя (уникальное) |
| email | varchar(254) | Email адрес |
| password | varchar(128) | Хэш пароля |
| first_name | varchar(150) | Имя |
| last_name | varchar(150) | Фамилия |
| role | varchar(20) | Роль (admin, moderator, user) |
| phone_number | varchar(20) | Номер телефона |
| avatar | varchar(100) | Путь к аватару |
| is_active | boolean | Активен ли пользователь |
| is_staff | boolean | Сотрудник (доступ к admin) |
| is_superuser | boolean | Суперпользователь |
| date_joined | timestamp | Дата регистрации |
| last_login | timestamp | Последний вход |

**Пример запроса:**
```sql
SELECT id, username, email, role, is_active, date_joined 
FROM accounts_customuser 
ORDER BY date_joined DESC;
```

### 2. ocr_language (Языки)

Поддерживаемые языки для распознавания

| Поле | Тип | Описание |
|------|-----|----------|
| id | bigint | Первичный ключ |
| code | varchar(10) | Код языка (ru, kk, en) |
| name | varchar(100) | Название языка |
| is_active | boolean | Активен ли язык |
| created_at | timestamp | Дата создания |
| updated_at | timestamp | Дата обновления |

**Пример запроса:**
```sql
SELECT * FROM ocr_language WHERE is_active = true;
```

### 3. ocr_ocrrequest (OCR Запросы)

Запросы на распознавание текста

| Поле | Тип | Описание |
|------|-----|----------|
| id | bigint | Первичный ключ |
| user_id | bigint | ID пользователя (FK) |
| language_id | bigint | ID языка (FK) |
| image | varchar(100) | Путь к изображению |
| recognized_text | text | Распознанный текст |
| status | varchar(20) | Статус (pending, processing, completed, failed) |
| confidence | decimal(5,2) | Точность распознавания (%) |
| processing_time | decimal(10,3) | Время обработки (сек) |
| error_message | text | Сообщение об ошибке |
| created_at | timestamp | Дата создания |
| updated_at | timestamp | Дата обновления |

**Пример запроса:**
```sql
SELECT 
    r.id,
    u.username,
    l.name as language,
    r.status,
    r.confidence,
    r.created_at
FROM ocr_ocrrequest r
JOIN accounts_customuser u ON r.user_id = u.id
JOIN ocr_language l ON r.language_id = l.id
ORDER BY r.created_at DESC
LIMIT 10;
```

### 4. ocr_ocrsettings (Настройки OCR)

Настройки системы распознавания

| Поле | Тип | Описание |
|------|-----|----------|
| id | bigint | Первичный ключ |
| key | varchar(100) | Ключ настройки |
| value | text | Значение |
| description | text | Описание |
| created_at | timestamp | Дата создания |
| updated_at | timestamp | Дата обновления |

**Пример запроса:**
```sql
SELECT key, value, description FROM ocr_ocrsettings;
```

### 5. dashboard_dashboardstats (Статистика)

Сохраненная статистика для дашборда

| Поле | Тип | Описание |
|------|-----|----------|
| id | bigint | Первичный ключ |
| total_users | integer | Всего пользователей |
| total_requests | integer | Всего запросов |
| successful_requests | integer | Успешных запросов |
| failed_requests | integer | Неудачных запросов |
| avg_confidence | decimal(5,2) | Средняя точность |
| date | date | Дата статистики |
| created_at | timestamp | Дата создания |

## 💾 Бэкап и восстановление

### Создание бэкапа

```bash
# Полный бэкап БД
sudo -u postgres pg_dump meta-university_db > backup_$(date +%Y%m%d_%H%M%S).sql

# Бэкап только структуры
sudo -u postgres pg_dump --schema-only meta-university_db > schema_backup.sql

# Бэкап только данных
sudo -u postgres pg_dump --data-only meta-university_db > data_backup.sql

# Бэкап в сжатом формате
sudo -u postgres pg_dump -Fc meta-university_db > backup_$(date +%Y%m%d_%H%M%S).dump
```

### Автоматический бэкап (cron)

```bash
# Создать скрипт бэкапа
cat > /root/backup_db.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/root/backups"
mkdir -p $BACKUP_DIR
sudo -u postgres pg_dump meta-university_db > $BACKUP_DIR/meta-university_$(date +%Y%m%d_%H%M%S).sql
# Удалить бэкапы старше 7 дней
find $BACKUP_DIR -name "meta-university_*.sql" -mtime +7 -delete
EOF

chmod +x /root/backup_db.sh

# Добавить в cron (каждый день в 3:00)
crontab -e
# Добавить строку:
0 3 * * * /root/backup_db.sh
```

### Восстановление из бэкапа

```bash
# Восстановление из SQL файла
sudo -u postgres psql meta-university_db < backup_20260211_034500.sql

# Восстановление из сжатого формата
sudo -u postgres pg_restore -d meta-university_db backup_20260211_034500.dump

# Полное пересоздание БД
sudo -u postgres psql << 'EOF'
DROP DATABASE meta-university_db;
CREATE DATABASE meta-university_db;
GRANT ALL PRIVILEGES ON DATABASE meta-university_db TO meta-university_user;
\q
EOF

sudo -u postgres psql meta-university_db < backup_20260211_034500.sql
```

## 📊 Полезные SQL запросы

### Статистика

```sql
-- Всего пользователей
SELECT COUNT(*) as total_users FROM accounts_customuser;

-- Активные пользователи
SELECT COUNT(*) as active_users FROM accounts_customuser WHERE is_active = true;

-- Всего OCR запросов
SELECT COUNT(*) as total_requests FROM ocr_ocrrequest;

-- Статистика по статусам
SELECT status, COUNT(*) as count 
FROM ocr_ocrrequest 
GROUP BY status;

-- Статистика по языкам
SELECT l.name, COUNT(r.id) as count
FROM ocr_language l
LEFT JOIN ocr_ocrrequest r ON l.id = r.language_id
GROUP BY l.name
ORDER BY count DESC;

-- Средняя точность распознавания
SELECT AVG(confidence) as avg_confidence 
FROM ocr_ocrrequest 
WHERE status = 'completed';

-- Топ пользователей по количеству запросов
SELECT 
    u.username,
    COUNT(r.id) as requests_count
FROM accounts_customuser u
LEFT JOIN ocr_ocrrequest r ON u.id = r.user_id
GROUP BY u.id, u.username
ORDER BY requests_count DESC
LIMIT 10;
```

### Очистка данных

```sql
-- Удалить старые запросы (старше 30 дней)
DELETE FROM ocr_ocrrequest 
WHERE created_at < NOW() - INTERVAL '30 days';

-- Удалить неактивные сессии
DELETE FROM django_session 
WHERE expire_date < NOW();

-- Очистить таблицу логов (если есть)
TRUNCATE TABLE django_admin_log;
```

### Проверка размеров

```sql
-- Размер базы данных
SELECT pg_size_pretty(pg_database_size('meta-university_db'));

-- Размер таблиц
SELECT 
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) as size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- Количество записей в таблицах
SELECT 
    schemaname,
    tablename,
    n_live_tup as row_count
FROM pg_stat_user_tables
ORDER BY n_live_tup DESC;
```

## 🔐 Безопасность

### Создание дополнительного пользователя (read-only)

```sql
-- Создать пользователя только для чтения
CREATE USER readonly_user WITH PASSWORD 'readonly_pass';
GRANT CONNECT ON DATABASE meta-university_db TO readonly_user;
GRANT USAGE ON SCHEMA public TO readonly_user;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO readonly_user;
```

### Смена пароля пользователя БД

## 🛠️ Обслуживание

### VACUUM и ANALYZE

```sql
-- Очистка и анализ БД
VACUUM ANALYZE;

-- Для конкретной таблицы
VACUUM ANALYZE ocr_ocrrequest;

-- Полная очистка (требует больше времени)
VACUUM FULL;
```

### Индексы

```sql
-- Просмотр индексов
\di

-- Создание индекса (пример)
CREATE INDEX idx_ocr_status ON ocr_ocrrequest(status);
CREATE INDEX idx_ocr_created ON ocr_ocrrequest(created_at);

-- Удаление индекса
DROP INDEX idx_ocr_status;
```

# ⚙️ Управление приложением на продакшене

## 🚀 Управление сервисами

### Статус приложения

```bash
# Проверить статус Django приложения
sudo systemctl status qazaqdana

# Проверить статус Nginx
sudo systemctl status nginx

# Проверить статус PostgreSQL
sudo systemctl status postgresql
```

### Перезапуск сервисов

```bash
# Перезапустить Django приложение
sudo systemctl restart qazaqdana

# Перезапустить Nginx
sudo systemctl restart nginx

# Перезапустить PostgreSQL
sudo systemctl restart postgresql

# Перезапустить всё
sudo systemctl restart qazaqdana nginx
```

### Остановка/Запуск сервисов

```bash
# Остановить приложение
sudo systemctl stop qazaqdana

# Запустить приложение
sudo systemctl start qazaqdana

# Включить автозапуск при загрузке сервера
sudo systemctl enable qazaqdana

# Отключить автозапуск
sudo systemctl disable qazaqdana
```

## 📊 Просмотр логов

### Логи приложения (Gunicorn)

```bash
# В реальном времени (live)
tail -f /var/www/qazaqdana/logs/gunicorn-error.log

# Последние 100 строк
tail -n 100 /var/www/qazaqdana/logs/gunicorn-error.log

# Логи доступа
tail -f /var/www/qazaqdana/logs/gunicorn-access.log

# Systemd логи
journalctl -u qazaqdana -f

# Последние 50 строк systemd логов
journalctl -u qazaqdana -n 50
```

### Логи веб-сервера (Nginx)

```bash
# Логи ошибок Nginx
tail -f /var/www/qazaqdana/logs/nginx-error.log

# Логи доступа Nginx
tail -f /var/www/qazaqdana/logs/nginx-access.log

# Системные логи Nginx
tail -f /var/log/nginx/error.log
```

### Очистка логов

```bash
# Очистить все логи приложения
cd /var/www/qazaqdana/logs
truncate -s 0 *.log

# Очистить только логи ошибок
truncate -s 0 gunicorn-error.log
```

## 🗄️ Работа с базой данных

### Подключение к PostgreSQL

```bash
# Войти в psql
sudo -u postgres psql

# Подключиться к БД проекта
sudo -u postgres psql -d qazaqdana_db

# От имени пользователя приложения
PGPASSWORD='QazaqDana2026!SecurePass' psql -U qazaqdana_user -d qazaqdana_db -h localhost
```

### Команды внутри psql

```sql
-- Список баз данных
\l

-- Список таблиц
\dt

-- Описание таблицы
\d accounts_customuser

-- Список пользователей
\du

-- Выход
\q
```

### Применение миграций Django

```bash
cd /var/www/qazaqdana
source venv/bin/activate

# Создать новые миграции
python manage.py makemigrations

# Применить миграции
python manage.py migrate

# Откатить миграцию
python manage.py migrate app_name migration_name

# Показать список миграций
python manage.py showmigrations
```

### Создание пользователя

```bash

```

## 🛠️ Устранение проблем

### Приложение не запускается

```bash
# 1. Проверить логи
journalctl -u qazaqdana -n 50

# 2. Проверить конфигурацию Gunicorn
cat /etc/systemd/system/qazaqdana.service

# 3. Попробовать запустить вручную
cd /var/www/qazaqdana
source venv/bin/activate
gunicorn --bind unix:/var/www/qazaqdana/gunicorn.sock danadjango.wsgi:application
```

### 502 Bad Gateway

```bash
# 1. Проверить, запущен ли Gunicorn
sudo systemctl status qazaqdana

# 2. Проверить наличие sock файла
ls -la /var/www/qazaqdana/gunicorn.sock

# 3. Проверить права доступа
sudo chown qazaqdana:www-data /var/www/qazaqdana/gunicorn.sock

# 4. Перезапустить сервисы
sudo systemctl restart qazaqdana nginx
```

### Ошибка подключения к БД

```bash
# 1. Проверить статус PostgreSQL
sudo systemctl status postgresql

# 2. Проверить .env файл
cat /var/www/qazaqdana/.env

# 3. Проверить подключение
PGPASSWORD='QazaqDana2026!SecurePass' psql -U qazaqdana_user -d qazaqdana_db -h localhost -c "SELECT 1;"
```

### Статика не загружается

```bash
# 1. Пересобрать статику
cd /var/www/qazaqdana
source venv/bin/activate
python manage.py collectstatic --clear --noinput

# 2. Проверить права
sudo chown -R qazaqdana:www-data /var/www/qazaqdana/staticfiles
sudo chmod -R 755 /var/www/qazaqdana/staticfiles

# 3. Перезапустить Nginx
sudo systemctl restart nginx
```
